# Decision Prototype Catalog（Interactive Demo Samples）

這個 Repo 放的是「可操作的一頁式 Demo」，用來快速對齊：
- 風格（視覺調性）
- 排版（資訊層級與版型）
- 敘事節奏（scroll / section flow）
- 方向判斷（go / no-go）

> 這些 Demo **不是最終交付成品**，也不是購物站模板。  
> 目的在於讓討論從「講不清楚」變成「滑一次就知道」。

---

## Demo 目錄

- **A｜OEM / 製造工廠**：`/factory-oem/`  
  工廠能力、製程可信度、合作流程導向  
- **B｜品牌故事 / 形象官網**：`/brand-story/`  
  品牌氣質、留白、視覺一致性導向  
- **C｜業務提案 / 快速決策**：`/sales-proposal/`  
  問題 → 解法 → 能力 → 下一步，資訊導向  
- **D｜產品概念 / MVP 驗證**：`/mvp-concept/`  
  這是什麼、為誰、價值是什麼，快速驗證導向  

---

## 如何使用（給看 Demo 的人）

1. 先開啟 Repo 的 GitHub Pages（或直接點 Demo 路徑）
2. 選擇最接近需求的場景 A/B/C/D
3. 看完後回覆：
   - 哪個最接近你要的感覺
   - 哪些區塊要保留 / 要拿掉
   - Hero 想用「影片」還是「圖片」

---

## 重要說明（避免誤解）

- 這些 Demo 用於前期溝通與方向對齊
- 不代表最終成品，也不代表已完成所有內容與細節
- 確認方向後，才進入正式製作與素材完善：
  - 文案、圖片、影片替換
  - RWD / 手機排版細節
  - 效能與載入速度
  - 動畫與互動強度調整

---

## 給協作方（若需要替換素材）

每個 Demo 資料夾內通常包含自己的 `README.md`（非工程人員版），會寫清楚：
- 影片放哪裡、檔名規則
- 圖片建議解析度
- `photos.json`（若有）如何修改
